# manual-test

## 目的
入力→プレビュー→出力→履歴までの主要フローを、仕様準拠で手動検証する。

## 対象仕様
- SPEC.md 7.1〜7.14
- SPEC.md 8.1 / 8.2 / 8.4

## 前提
- `npm ci` 実行済み
- `npm run dev` でアプリ起動済み
- 最新版のモダンブラウザを使用

## テスト観点

### 1) 入力即時反映
1. 上枠・下枠テキストを変更し、プレビューが即時更新されること。
2. 日付変更で中央表示が即時更新されること。
3. 書体・文字サイズ倍率・色・枠線太さ/色の変更が即時反映されること。

### 2) 日付フォーマット
1. 西暦/和暦切替で表示が切り替わること。
2. 区切り文字のプリセットと自由入力が表示に反映されること。

### 3) 出力
1. PNGダウンロードが成功すること。
2. PNG透過背景ON/OFFで背景状態が変わること。
3. SVGダウンロードが成功し、印影構造（円・横線2本・3領域テキスト）を保持すること。

### 4) 履歴
1. 履歴保存を10件超で実行し、古い履歴から削除されること。
2. 履歴選択でフォームとプレビューが復元されること。

### 5) 非機能
1. 入力変更時の再描画が体感で遅延なく動作すること（SPEC 8.1）。
2. Chrome / Edge / Firefox / Safari で主要操作が完了できること（SPEC 8.2）。
3. ブラウザ開発者ツールのNetworkで外部通信が発生しないこと（SPEC 7.14, 8.4）。
